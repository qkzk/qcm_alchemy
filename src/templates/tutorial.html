Comment utiliser ce site ?

Pour l'élève

Depuis l'accueil choisir le lien Élève. Saisir vos nom et prénom ainsi que le numéro du QCM transmis par votre enseignant.

Ensuite sélectionnez les bonnes réponses et n'oubliez pas de cliquer sur le bouton Envoyer.

Mode anti-triche

Afin de rendre l'évaluation un peu plus sérieuse, vous ne pouvez pas quitter la fenêtre. Dès que vous ouvrez un nouvel onglet ou quittez la fenêtre, vos réponses sont enregistrées et vous ne pouvez plus progresser.

Pour l'enseignant

Une petite vidéo de présentation.

L'objectif de ce site est de créer des évaluations rapidement, les partager simplement, consulter les réponses et les scores sans devoir compter les points.

Les enseignants doivent s'enregistrer avec un email - pas encore de confirmation... c'est prévu, on ne sait pas quand.

Pas encore de désinscription, cela ne devrait pas tarder.

Une fois enregistrés, ils peuvent se connecter. Un mot de passe égaré peut être réinitialisé avec une clé envoyée par email. Celle-ci est valide une heure.

Ce mot de passe est haché et personne n'y a accès.

Ensuite :

Depuis l'accueil choisir le lien Professeur. Là vous pouvez choisir d'envoyer un nouveau QCM ou consulter les résultats de vos précédents QCM.

Pour envoyer un nouveau QCM, sélectionnez un fichier compatible au format .md - voir plus bas - et, une fois uploadé, transmettez son numéro à vos élèves qui en auront besoin pour débuter le QCM.

Si vous avez perdu ce numéro, vous pouvez consulter la liste de vos QCM disponibles depuis le menu enseignant.

Vous pouvez visualiser votre QCM après son dépôt et avant de le partager à vos élèves.

Vos QCM sont personnels, seul l'auteur peut les consulter. Il est en le propriétaire. Nous ne sommes donc pas responsable du contenu qu'il contient, ne pouvant même pas le consulter.

Une fois l'évaluation terminée, depuis le menu Professeur, choisissez "Liste des QCMs" et vous verrez la liste des QCMs disponibles.

De là, vous pouvez consulter les travaux des élèves et visualiser leurs réponses. Vous pouvez aussi simplement consulter les scores obtenus : 1 bonne réponse = 1 point, pas de barème encore...

L'enseignant peut exporter les scores et les réponses au format csv. Le csv rendu est un peu lourd, en effet il contient les réponses à toutes les questions. Les tableurs l'ouvrent sans difficulté. Les deux premières colonnes contiennent le nom et le score, si c'est tout ce dont vous avez besoin.

Attention, tous les QCM sont effacés ainsi que les comptes élèves et leurs notes après 48h. Pensez à récupérer vos résultats avant.

Format de fichiers QCM

Vous pouvez consulter un exemple ici. Pour résumer :

Du markdown

Les titres des parties sont précédés de ##

Les questions sont de deux natures : choix multiple ou zone de texte. Elles sont précédées de ###

Pour les questions à choix multiple : Les propositions sont précédées de - [ ] pour les mauvaises réponses et - [x] pour les bonnes réponses.

Une ou plusieurs bonnes réponses par question. Mais le candidat ne peut choisir qu'une réponse.

Pour les questions avec une zone de texte, il faut une seule ligne après la question au format - [t]

Des images peuvent être incluses, mais elles doivent être hébergées en ligne.

Vous pouvez inclure du code, ou du LaTeX, ainsi que vous pouvez le voir dans l'exemple.

Par exemple :

$\int_1^2x dx = \frac{3}{2}$ est rendu en LaTeX : $\int_1^2x dx = \frac{3}{2}$

`1 + 2 == 3` est présenté comme du code : 1 + 2 == 3.

Le parser est assez rudimentaire, vous devez respecter scrupuleusement le format. Partez de l'exemple ici.

D'où vient l'idée ?

Google form + google classroom pour l'aspect échange facile de documents, récupération des notes en CSV, panneau avec les notes. Mais pas de code, pas de latex peu de formattage, impossible de créer depuis un fichier texte.

Tous les sites de QCM/Form/Quizz qui ont beaucoup d'atouts mais présentent tous un défaut majeur :

Pas d'export en .pdf ou pas de possibilité de rendre du LaTeX / code ou pas de possibilité de créer depuis un fichier source...

Ceux de pronote sont quasi parfaits mais : pas de code, pas de latex, pas de création depuis un fichier.

Ceux de doctools sont bien plus riches mais ne permettent pas de créer depuis un fichier.

En LaTex il existe des outils qui font sensiblement la même chose, je n'ai jamais vraiment creusé mais vous pouvez chercher automultiplechoice.
